## 5. Insights and Learnings

- **Empirical verification of literal claims in user-facing docs is a hard requirement, not a nice-to-have.** Two NO-GOs in two consecutive PRs from the same root cause is a strong signal — and a third recent incident (downstream PR #29's `@validates` docstring) shows it's recurring across forks. The rule shipped in #146/#147 names a *review activity* (run the literal claim and check) rather than just naming a bug category. Without explicit naming, reviewers consistently approve doc PRs by reading them.
- **CommonMark backtick parsing is line-sensitive.** Patterns like `` (`Ctrl+` `)` `` only "work" because line breaks prevent backtick pairing across the text. Once the pattern is on a single line, the backticks pair into code spans containing spaces and MD038 fires. The latent bug is undetectable until someone restructures the surrounding text.
- **`gh repo view --json owner` returns `{id, login}` only.** No `type` field. Use `gh repo view --json isInOrganization --jq '.isInOrganization'` for org-vs-user detection (boolean output, native gh CLI, no REST API call). The REST API equivalent (`gh api repos/<owner>/<repo> --jq '.owner.type'`) returns the literal `Organization`/`User` string but requires the extra round-trip.
- **The framework's auto-* workflows do not call Claude.** `auto-fix.yml`, `auto-review.yml`, and `auto-triage.yml` are trigger scaffolding that post comments inviting the user to invoke `/review-pr` etc. locally. The framework does not currently run Claude in CI at all. Downstream forks that want Claude-in-CI are on their own to wire it up.
- **GitHub has two separate secret stores** — Actions secrets (workflow runs only) and Codespaces secrets (injected as env vars into Codespaces). Setting `ANTHROPIC_API_KEY` as an Actions secret will NOT make it available to Claude Code in a Codespace. The two stores are independent and serve different purposes.
- **Project board #13 is the working board for `vibeacademy/agile-flow-gcp`**, not project #4. Project #4 is for the non-GCP fork (`vibeacademy/agile-flow`). When adding tickets to the board, query an existing peer ticket's `projectItems` to confirm — don't infer from `gh project list`.
- **`/log-session` writes a journal to local disk but does not ship it.** Per #137 (filed this session), session journals routinely bypass the PR rule because the ceremony is heavier than the content warrants. Until #137 (`/ship-docs`) lands, the journal needs a Quick Fix branch + PR by hand.
