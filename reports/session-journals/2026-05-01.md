## 5. Insights and Learnings

- **TTY-based context inference is unreliable across hosting platforms.**
  Codespaces `postCreateCommand` has a TTY despite running as a "background"
  provisioning step. The #97 fix's `[ ! -t 0 ]` guard didn't fire there.
  Use platform-specific env vars (`CODESPACES`, `CI`, `GITHUB_ACTIONS`)
  when behavior should branch on hosting context. (Surfaced via #102.)

- **`set -o pipefail` + `grep` is a fail-closed pattern by default.**
  `gh auth status | grep ... | sed ...` will kill a script on unparsable
  state because grep returns 1 on no-match. Pipelines that should
  fail-soft need explicit `|| true` at the end. The hook on `main` had
  this latent bug for weeks; #102's tests exposed it. (Same class as
  #45's silent-skip-cascade lesson.)

- **BSD awk vs GNU awk diverge on multi-line `-v` strings.**
  BSD awk (macOS default) errors with "newline in string" when given
  a multi-line value via `-v`. The portable pattern is: write the
  multi-line value to a temp file, read it inside the awk script with
  `while ((getline line < file) > 0)`. Same shape as #74's portable
  `tac`-replacement-via-`awk` pattern.

- **Same UI-panel constraint applies to multiple GitHub project
  workflows.** #86 established that `updateProjectV2Workflow` doesn't
  exist in the GraphQL API — Auto-Move-to-Done can only be enabled via
  the web UI. The Auto-add-to-project workflow is in the same panel
  with the same limitation. The framework's "Manual UI toggle" pattern
  is the right accommodation; #120 extended it to cover both workflows
  in one place. Whenever the framework documents a UI workaround for a
  GitHub-API limitation, check whether other workflows in the same
  panel have the same constraint.

- **Auto-add-to-project does not backfill.** Issues created BEFORE the
  workflow is enabled stay off the board permanently unless manually
  added. The natural Phase 4 ordering put issue creation before
  workflow toggles, stranding 9 issues in the kira-kim-11 dry-run.
  Re-sequencing was the fix; backfill instructions were the safety
  net. (Surfaced in #120.)

- **Codespaces auto-injects `GITHUB_TOKEN` (a `ghu_` user-to-server
  token) but not `GH_TOKEN`.** `gh` prefers `GH_TOKEN` if both exist,
  so configuring a `GH_TOKEN` Codespaces user secret with broader
  scopes is the supported workaround. The auto-injected `GITHUB_TOKEN`
  is refresh-impossible by design — gh doesn't own it. This is not
  GitHub's bug; it's the documented behavior. The framework needs to
  route around it; see #102's Step 4 Codespaces branch.

- **Org-hosted workshop architecture is meaningfully simpler for
  attendees than the personal-account-fork model.** Decision locked
  today: `vibeacademy/<handle>` repos, Codespaces org secrets
  (Anthropic + Neon + GCP), facilitator absorbs cost, per-attendee
  Neon project, `gh repo transfer` post-workshop. The model collapses
  attendee setup from "generate PAT + configure 2 secrets + Anthropic
  account login" to "open Codespace + run `claude`." But it pushes
  significant work to the facilitator (one-time provisioning, Neon
  org, GCP project, billing) — small/medium cohorts only. Documented
  in `agile-flow-meta:docs/workshops/gcp-architecture-may-2026.md`.

- **Vet sibling-session tickets before merging into the priority
  flow.** Another Claude session created #118, #119, #120 from the
  raw handoff without rescoping under today's architecture decisions.
  Tickets were technically correct but missed cross-references and
  context. Solution: append rescoping comments + labels + board
  placement on receipt. Pattern: when a parallel session produces
  work product, treat it as a draft to be reviewed, not a finished
  artifact.

- **5 PRs in one session is a sustainable cadence for small-mechanical
  tickets if you've already done the architecture decision.** Round 1
  worked because the architecture was locked first (in the #92 meta
  PR), giving each ticket a clear scope boundary and a defined
  reference point for "is this in scope?" Trying the same cadence on
  Round 2 (architectural plumbing) without the up-front decision work
  would have been chaotic.
